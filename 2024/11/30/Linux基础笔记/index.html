基础命令

创建目录(directory)：mkdir (目录名)

删除目录(directory)：rm -rf (路径/目录名)

查找目录路径：find / -type d -name dir142 -exec rm -r {} ;

查找文件路径：find / -name 文件名

创建文件(file)：touch (文件名)

创建隐藏文件：touch .(文件名) (隐藏文件名以英文点号.开头即可)

复制文件：cp file(文件名） /root/目录

移动文件：mv (文件名) /root/(目录)

进入目录：cd (路径)

查询当前目录下的文件：ll (ls -l)

查看文件内容：cat (路径/文件名)

/// DHCP服务 配置为192.168.152.0/24网段自动分配IP地址，分配的IP地址段为：192.168.152.200-192.168.152.210，分配的网关为：192.168.152.2，分配的DNS服务器地址：192.168.152.2

}
保存退出

/// DNS服务

实现以下域名的正向和反向解析：www.abc.com解析为：192.168.1.100，ftp.abc.com解析为：192.168.1.200，mail.abc.com为邮件交换器【请安装好bind-utils，否则测试不通过】

安装 bind-utils
yum install bind bind-utils -y
编辑主配置文件
vi /etc/named.conf
在最下面添加以下内容，配置正向和反向区域
zone “abc.com” IN {
type master;
file “/var/named/forward.abc.com”;
allow-update { none; };
};

zone “1.168.192.in-addr.arpa” IN {
type master;
file “/var/named/reverse.abc.com”;
allow-update { none; };
};
创建正向解析区文件 touch /var/named/forward.abc.com
编辑文件 vi /var/named/forward.abc.com
添加以下内容：
$TTL 86400
@ IN SOA ns1.abc.com. root.abc.com. (
2024053101 ; Serial
3600 ; Refresh
1800 ; Retry
604800 ; Expire
86400 ; Minimum TTL
)

@ IN NS ns1.abc.com.

ns1 IN A 192.168.1.100
www IN A 192.168.1.100
ftp IN A 192.168.1.200
mail IN MX 10 mail.abc.com.
mail IN A 192.168.1.100
创建反向解析区文件 touch /var/named/reverse.abc.com
编辑文件 vi /var/named/reverse.abc.com
添加以下内容：
$TTL 86400
@ IN SOA ns1.abc.com. root.abc.com. (
2024053101 ; Serial
3600 ; Refresh
1800 ; Retry
604800 ; Expire
86400 ; Minimum TTL
)

@ IN NS ns1.abc.com.

ns1 IN A 192.168.1.100
100 IN PTR www.abc.com.
200 IN PTR ftp.abc.com.
100 IN PTR mail.abc.com.
接下来启动 BIND 服务
systemctl start named
没报错就对了
(此步骤可以不用) 检查 BIND 配置文件语法
named-checkconf

named-checkzone abc.com /var/named/forward.abc.com

named-checkzone 1.168.192.in-addr.arpa /var/named/reverse.abc.com
上面的后两条命令输入并回车后，都出现OK即表示成功，可以提交检查

///WEB服务(Apache)虚拟主机

通过修改虚拟主机端口，不能修改虚拟主机名称，同时实现通过浏览器打开http://IP:8081，可以显示Hello 475 8081! 打开http://IP:8082，可以显示Hello 475 8082!（注意IP为Linux本机的IP，Hello 475 8081!或Hello 475 8082!注意大小写、空格和符号）

1.先安装Apache
yum install -y httpd
2.启动Apache
systemctl start httpd
3.配置虚拟主机
vi /etc/httpd/conf.d/8081.conf 添加以下内容：
Listen 8081
<VirtualHost *:8081>
DocumentRoot /var/www/html-8081
ErrorLog /var/log/httpd/error_8081.log
CustomLog /var/log/httpd/access_8081.log combined
<Directory /var/www/html-8081>
Require all granted


（注意复制粘贴时上面会少几个字母）

vi /etc/httpd/conf.d/8082.conf 再给8082添加以上同样内容、记得修改成8082

4.创建网站目录和文件
mkdir -p /var/www/html-8081
mkdir -p /var/www/html-8082
5.使用vi编辑器编辑
vi /var/www/html-8081/index.html
输入：Hello 667 8081！
vi /var/www/html-8082/index.html
输入：Hello 667 8082！
6.重启
systemctl restart httpd
7.提交检查即可
如出现启动Apache后，systemctl status httpd是actice，配置完后变红，解决办法如下：
vi /etc/httpd/conf/httpd.conf 找到Listen80改为Listen 8080
setenforce 0
然后输入systemctl restart httpd，再检查
